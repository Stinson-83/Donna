Color

Four locked values carry the identity. Everything else is a derived scale in the same family · warm, paper-based, never cool. Color is rare. One accent moment per screen.

Locked · core four

The page. Never pure white. Warm enough to feel lit from the side.

Raised surfaces. One step above paper. Pairs with hairlines, not shadows.

The rule
When to use color

Rust is reserved for the one thing that matters most on the screen. A CTA. A single word she’s emphasizing. The live status of a meaningful moment. Never two rusts on one screen.

Signal colors appear only on states · a chip, a dot, a line. Never as backgrounds. Never as decoration.

When to use black

Everything else. Headings, body, labels, icons. Black (ink) is the default voice. Color is the exception that earns attention because it’s rare.

If you’re reaching for color to make something legible, the layout is wrong.

The rule
What is serif for

EB Garamond italic is used for anything the user should feel, not just read. Page titles, section leads, card titles, her own words when she addresses you directly, quoted memory, the hero of an empty state.

Serif is always italic. Upright serif is not Donna.

What is sans for

Red Hat Text handles the mechanics. Body copy, labels, buttons, inputs, metadata, timestamps, navigation. Anything a user reads to act.

Sans is never italic in the UI. Never. Italic sans reads as apology.

When headings are in color

Almost never. The single allowed case: one word or phrase inside a heading gets rust to mark its weight · a name, an outcome, a promise. Never the whole heading.

Example: A quieter kind of AI.   or   Today with Maya.

When headings are in black

Default. Every page title, section title, card title is ink. The serif italic is already doing emotional work · color would be redundant.

Weights

Serif: 400 (regular italic) for everything. 500 exists for emphasis within a paragraph but is almost never needed.

Sans: 400 for body, 500 for buttons and labels, 600 for rare cases where a specific word must carry. Never 700 in product UI.

The rule
Inside a card

Padding: space-5 (24) on mobile, space-6 (32) on desktop.

Gap between a card's title and body: space-3 (12). Between body and footer/meta: space-4 (16). Never tighter.

Between cards

Grids: space-5 (24) gutter, or a 1px hairline in place of gap for a denser, editorial feel.

Stacked cards: space-4 (16) if related, space-6 (32) if they are distinct thoughts.

Between sections

space-8 (64) is the minimum between two sections that belong to the same page. space-9 (96) between chapters. space-10 (128) when you want the user to breathe · hero to first content, for example.

Page gutters

Mobile: 20 px left/right. Desktop: 32 px minimum.

Dashboard max width: 1200 px. Reading content: 640 px. Never full-bleed except for hero images.

The vertical rhythm

Headings always get more space above than below. A card's title sits closer to its body than to the card above. White space belongs to the thing that follows, not the thing that came before.

If a screen feels crowded, the fix is always to add space, never to shrink type.

Motion

Barely there. Donna doesn’t announce herself · she appears. Most motion is a slow fade, a gentle translate, a character-by-character type reveal. No bounces. No spring overshoot. Nothing that calls attention to the animation itself.

Durations
Fast · 160ms
Hovers, button states, tiny toggles.
Normal · 280ms
Card entrance, dropdown, menu, toast.
Slow · 480ms
Page transitions, hero reveals, her messages.
The rule
Easing

Default: cubic-bezier(0.22, 1, 0.36, 1) · a soft ease-out. Things arrive, they don’t bounce.

Page-level reveals: cubic-bezier(0.2, 0.8, 0.2, 1) · editorial, turning-a-page feel.

Never linear. Never spring.

Her messages

When Donna speaks, text appears with a gentle 60ms-per-word fade-in (not a typewriter). Never faster. It should feel like she’s thinking, not performing.
