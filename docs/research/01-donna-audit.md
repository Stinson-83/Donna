# Donna Codebase Audit (factual baseline)

Date: 2026-04-23. Source: donna-audit agent trace.

## BRAIN loop (`donna_runtime/brain.py`, `runner.py`)
- Entry `donna_turn(state, config)` → `runner.traced_donna_turn()` → wraps `claude_agent_sdk.query()` (runner.py:61).
- `max_turns` default = 3 (config.py:73).
- Parallel tool calls: handled natively by SDK, recorded sequentially in trace.
- Errors: TimeoutError + generic Exception caught; resume session ID cleared on failure.
- Resume retry: if a resumed session emits zero tool calls, retries fresh (brain.py:56–63).

## System prompt (`prompt.py`, `context_builder.py`)
- `_DONNA_CORE` ~5,400 chars (identity, voice, safety floors, recovery).
- Terminator contract: every turn ends with `send_burst()` or `stay_silent()`.
- Two tiers: STAGE_0 (no tools), STAGE_0_5 (tools via MCP).
- **No `cache_control` directives anywhere.** Prompt caching not configured.
- Living Profile DEPRECATED: `load_living_profile()` exists but `prompt.py:261` explicitly `del living_profile`. Per-turn runtime context regenerated by `context_builder.render_turn_context()`, capped at 5000 chars.

## Tools (`donna_runtime/tools.py`)
12 runtime tools, ~375 tokens total catalog weight. Most have when-to-use; when-NOT-to-use clauses weak or absent:
- `read_tracker` (51 chars) — no guidance.
- `recall_graph` (15 chars) — trivial.
- Others (80–300 chars) — adequate.
- Terminators (`send_burst`, `stay_silent`) well-described.

Backend memory tools: 16 total (3 retrieval, 4 list, 3 synthesis/read, 5 write).

## Hooks
- Runtime (`donna_runtime/hooks.py`): `pre_tool_hook` / `post_tool_hook` → trace recording + LangSmith.
- Memory (`backend/memory/hooks/__init__.py`): after every `send_burst`, 4 async fire-and-forget hooks: `save_chat_messages`, `record_episode` (Supermemory), `ingest_to_graph` (Graphiti/FalkorDB), `extract_user_facts`.
- Drained at end of multi-turn session.

## Temporal brief
`backend/memory/synthesis/temporal_brief.py` (895 lines). Five implementations (RECENT_CONTEXT, WINDOWED_TIMELINE, ATTENTION_WEIGHTED, CLAUDE_SYNTHESIS, COMPILED_STATE). Invoked only by nightly job or explicit `refresh_situation_brief` tool. **Does NOT violate "no pre-generated situational brief" rule** — brief is stored, model must call `read_situation_brief` to pull it.

Eval results on 18 archetypes + 8 overload variants: WINDOWED_TIMELINE avg 72.1, ATTENTION_WEIGHTED 70.9, COMPILED_STATE 69.4.

## Cost posture (inferred)
No caching + Sonnet default + per-turn context rebuild ≫ $0.01 reactive target. Primary cost levers missing:
1. No `cache_control` on `_DONNA_CORE` (5,400 chars = ~1,400 stable tokens).
2. No cache on tool catalog.
3. Sonnet 4.6 ~5× Haiku 4.5 input, ~4× output.
